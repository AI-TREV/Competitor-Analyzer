<objective>
## Goal
Create the SQLite schema (8 tables), migration runner, and seed data so `npm run db:setup` produces a working database with all competitors, stores, and internal SKU placeholders.

## Purpose
Every subsequent phase depends on this foundation. This plan incorporates all forensic fixes identified in pre-execution review (session 2026-05-04):
- `internal_sku` table (missing from original spec, fatal to Phase 5 reporting)
- `internal_price_history` table (required for historical gap analysis in Phase 5)
- `observed_at` has no DEFAULT — scraper must supply actual HTTP response time
- `competitor_price_observation` has UNIQUE constraint preventing duplicate observations
- `screenshot_path` has CHECK constraint enforcing relative-only paths
- `competitor_product` has UNIQUE constraint preventing duplicate product rows across runs
- `competitor_store` tracks `last_scraped_at` + `last_scrape_status` for staleness detection
- 5 indexes (up from 4) including price history index for Phase 5 gap queries
- `scrape_run_id` documented as UUID throughout
- `exception_queue.scrape_run_id` FK has ON DELETE SET NULL

## Output
- `migrations/001-initial-schema.sql` — 8 tables + 5 indexes + FK constraints + CHECK constraints
- `src/config/database.js` — better-sqlite3 connection + migration runner (absolute path resolution)
- `src/config/seed.js` — idempotent seed: 2 competitors, 12 stores, 4 internal SKU stubs
- `package.json` — better-sqlite3 added, `db:setup`, `db:migrate`, `db:seed` scripts added
</objective>

  <files>apps/competitor-scraper/package.json, apps/competitor-scraper/migrations/001-initial-schema.sql</files>
  <action>
    1. CREATE `apps/competitor-scraper/package.json` — this file does not yet exist: <!-- audit-added: MH-1 -->
       ```json
       {
         "name": "competitor-scraper",
         "version": "0.1.0",

       ```
       `"type": "module"` is REQUIRED — database.js and seed.js use ES module syntax (import/export). Without it Node.js treats .js files as CommonJS and throws SyntaxError.
    2. Run `cd apps/competitor-scraper && npm install` to compile and install the better-sqlite3 native addon. <!-- audit-added: MH-2 -->
    3. Create `apps/competitor-scraper/migrations/001-initial-schema.sql` with all 8 tables in FK-dependency order:

    **Table 1: competitor** (unchanged from spec)
    ```sql
    CREATE TABLE IF NOT EXISTS competitor (
      competitor_id INTEGER PRIMARY KEY AUTOINCREMENT,
      name TEXT NOT NULL UNIQUE,
      region TEXT NOT NULL DEFAULT 'Auckland',
      website TEXT NOT NULL,
      created_at TEXT NOT NULL DEFAULT (datetime('now')),
      updated_at TEXT NOT NULL DEFAULT (datetime('now'))
    );
    ```

    **Table 2: competitor_store** (adds last_scraped_at + last_scrape_status for dashboard staleness detection)
    ```sql
    CREATE TABLE IF NOT EXISTS competitor_store (
      store_id INTEGER PRIMARY KEY AUTOINCREMENT,
      competitor_id INTEGER NOT NULL REFERENCES competitor(competitor_id),
      store_name TEXT NOT NULL,
      store_url TEXT,
      region TEXT NOT NULL DEFAULT 'Auckland',
      active INTEGER NOT NULL DEFAULT 1,
      last_scraped_at TEXT,
      last_scrape_status TEXT NOT NULL DEFAULT 'never' CHECK(last_scrape_status IN ('never', 'success', 'failed', 'partial')), <!-- audit-added: SR-1 -->
      created_at TEXT NOT NULL DEFAULT (datetime('now'))
    );
    ```

    **Table 3: internal_sku** (required for Phase 5 price gap reporting)
    ```sql
    CREATE TABLE IF NOT EXISTS internal_sku (
      sku_id TEXT PRIMARY KEY,
      name TEXT NOT NULL,
      egg_method TEXT NOT NULL DEFAULT 'free-range',
      pack_size INTEGER,
      current_price REAL,
      updated_at TEXT NOT NULL DEFAULT (datetime('now')),
      created_at TEXT NOT NULL DEFAULT (datetime('now'))
    );
    ```

    **Table 4: internal_price_history** (NEW — required for Phase 5 historical price gap analysis; single current_price would lose history when operator updates their price)
    ```sql
    CREATE TABLE IF NOT EXISTS internal_price_history (
      history_id INTEGER PRIMARY KEY AUTOINCREMENT,
      sku_id TEXT NOT NULL REFERENCES internal_sku(sku_id),
      price REAL NOT NULL,
      effective_from TEXT NOT NULL DEFAULT (datetime('now')),
      notes TEXT,
      created_at TEXT NOT NULL DEFAULT (datetime('now'))
    );
    ```

    **Table 5: competitor_product** (adds UNIQUE constraint — prevents duplicate product rows across scrape runs)
    ```sql
    CREATE TABLE IF NOT EXISTS competitor_product (
      competitor_product_id INTEGER PRIMARY KEY AUTOINCREMENT,
      competitor_id INTEGER NOT NULL REFERENCES competitor(competitor_id),
      competitor_name TEXT NOT NULL,
      brand TEXT,
      egg_method TEXT DEFAULT 'free-range',
      size TEXT,
      pack_size INTEGER,
      mapped_sku TEXT REFERENCES internal_sku(sku_id),
      match_confidence REAL DEFAULT 0.0,
      created_at TEXT NOT NULL DEFAULT (datetime('now')),
      updated_at TEXT NOT NULL DEFAULT (datetime('now')),
      UNIQUE(competitor_id, competitor_name, pack_size) -- NOTE: SQLite treats NULL as distinct in UNIQUE — two rows with pack_size=NULL will not conflict. Phase 2/3 normalizer must handle this. <!-- audit-added: SR-2 -->
    );
    ```

    **Table 6: scrape_run** (unchanged; document UUID requirement in comment)
    ```sql
    -- IMPORTANT: scrape_run_id MUST be crypto.randomUUID() — never sequential integer or timestamp
    CREATE TABLE IF NOT EXISTS scrape_run (
      scrape_run_id TEXT PRIMARY KEY,
      started_at TEXT NOT NULL DEFAULT (datetime('now')),
      completed_at TEXT,
      status TEXT NOT NULL DEFAULT 'running' CHECK(status IN ('running', 'completed', 'failed', 'partial')), <!-- audit-added: SR-1 -->
      total_observations INTEGER DEFAULT 0,
      errors INTEGER DEFAULT 0,
      notes TEXT
    );
    ```

    **Table 7: competitor_price_observation** (removes DEFAULT from observed_at; adds UNIQUE constraint; adds CHECK on screenshot_path)
    ```sql
    CREATE TABLE IF NOT EXISTS competitor_price_observation (
      observation_id INTEGER PRIMARY KEY AUTOINCREMENT,
      competitor_product_id INTEGER NOT NULL REFERENCES competitor_product(competitor_product_id),
      store_id INTEGER NOT NULL REFERENCES competitor_store(store_id), -- NOT NULL: nullable store_id silently bypasses UNIQUE(competitor_product_id, store_id, scrape_run_id) in SQLite (NULL≠NULL) <!-- audit-added: MH-4 -->
      scrape_run_id TEXT NOT NULL REFERENCES scrape_run(scrape_run_id),
      observed_at TEXT NOT NULL, -- NO DEFAULT: scraper stamps actual HTTP response time
      price REAL,
      member_price REAL,
      unit_price REAL,
      stock_status TEXT DEFAULT 'unknown',
      promotion_label TEXT,
      source_url TEXT NOT NULL,
      screenshot_path TEXT CHECK(
        screenshot_path IS NULL OR
        (screenshot_path NOT LIKE '/%' AND screenshot_path NOT LIKE '_:\%')
      ), -- RELATIVE path only: "{run_id}/{filename}.png". Resolve to absolute with SCREENSHOT_DIR at read time.
      raw_json_snapshot TEXT,
      created_at TEXT NOT NULL DEFAULT (datetime('now')),
      UNIQUE(competitor_product_id, store_id, scrape_run_id)
    );
    ```

    **Table 8: exception_queue** (exception_type comment documents valid taxonomy values)
    ```sql
    CREATE TABLE IF NOT EXISTS exception_queue (
      exception_id INTEGER PRIMARY KEY AUTOINCREMENT,
      competitor_product_id INTEGER REFERENCES competitor_product(competitor_product_id),
      scrape_run_id TEXT REFERENCES scrape_run(scrape_run_id) ON DELETE SET NULL,
      exception_type TEXT NOT NULL, -- MATCH_CONFIDENCE_LOW | PARSE_FAILED | PRICE_ANOMALY | JSON_INVALID | SITE_BLOCKED | SKU_REF_ERROR
      description TEXT,
      raw_data TEXT,
      status TEXT NOT NULL DEFAULT 'pending' CHECK(status IN ('pending', 'resolved', 'skipped')), <!-- audit-added: SR-1 -->
      resolved_at TEXT,
      resolved_by TEXT,
      created_at TEXT NOT NULL DEFAULT (datetime('now'))
    );
    ```

    4. After all CREATE TABLE statements, add these 5 indexes:
    ```sql
    CREATE INDEX IF NOT EXISTS idx_obs_scrape_run ON competitor_price_observation(scrape_run_id);
    CREATE INDEX IF NOT EXISTS idx_obs_product ON competitor_price_observation(competitor_product_id);
    CREATE INDEX IF NOT EXISTS idx_obs_observed_at ON competitor_price_observation(observed_at);
    CREATE INDEX IF NOT EXISTS idx_exc_scrape_run ON exception_queue(scrape_run_id);
    CREATE INDEX IF NOT EXISTS idx_price_history_sku ON internal_price_history(sku_id, effective_from);
    ```

    Avoid: splitting CREATE TABLE and CREATE INDEX into separate files — schema must be atomic in one file.

  <name>Task 2: Create database connection and migration runner</name>
  <files>apps/competitor-scraper/src/config/database.js</files>
  <action>
    Create `apps/competitor-scraper/src/config/database.js` as an ES module with two responsibilities: DB connection singleton and migration runner.

    **Connection:**
    ```js
    import Database from 'better-sqlite3';
    import path from 'path';
    import { fileURLToPath } from 'url';

    const __dirname = path.dirname(fileURLToPath(import.meta.url));

    // Resolve to absolute at startup — never cwd-relative at call time
    const DB_PATH = path.resolve(
      process.env.DATABASE_PATH ?? path.join(__dirname, '../../data/competitor-scraper.db')
    );

    export const db = new Database(DB_PATH);
    db.pragma('journal_mode = WAL');
    db.pragma('foreign_keys = ON');
    ```

    **Migration runner** (invoked when `process.argv[2] === 'migrate'`):
    - Resolve migrations directory relative to this file (NOT process.cwd()): <!-- audit-added: MH-3 -->
      `const MIGRATIONS_DIR = path.join(path.dirname(fileURLToPath(import.meta.url)), '../../migrations');`
    - Read all `.sql` files from MIGRATIONS_DIR in sorted filename order (`fs.readdirSync(MIGRATIONS_DIR).filter(f => f.endsWith('.sql')).sort()`)
    - Execute each as a transaction: `db.exec(sql)` wrapped in `db.transaction(() => ...)()`
    - Log each file name as it runs
    - Log final table count from `sqlite_master` on success
    - Exit 0 on success, exit 1 on error

    **Ensure `data/` directory exists** before opening DB (use `fs.mkdirSync(dir, { recursive: true })`).

    Avoid: executing raw shell commands, using synchronous file reads in the hot path (migrations only run on startup).
  </action>
